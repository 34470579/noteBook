# gdb

## 1、调试的基本思路

分析现象->假设错误原因->产生新的现象去验证假设

## 2、单步执行和跟踪函数调用

|命令|描述|
|-|-|
|backtrace(bt)|查看各级函数及调用|
|finish|连续运行到当前函数返回为止，然后停下来等待|
|frame(f)|选择帧栈|
|info(i) locals|查看当前帧栈局部变量的值|
|list(l)|列出源码，每次10行|
|next(n)|执行下一行语句|
|print(p)|打印表达式的值|
|quit(q)|退出gdb|
|set var|设置变量的值|
|start|开始执行程序|
|step(s)|执行下一行语句，如果有函数调用，进入函数|
